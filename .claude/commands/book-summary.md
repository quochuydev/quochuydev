Research and create a comprehensive book summary for: $ARGUMENTS

## Output Structure

Create a markdown file with:

### 1. Book Overview
- Title, Author, Publication Year
- Genre/Category
- One-sentence thesis (the core argument)
- Why this book matters (context and relevance)

### 2. The Big Idea
- 2-3 paragraphs explaining the central concept in plain language
- What problem does the book solve?
- Who should read this book?

### 3. Key Takeaways (5-7 points)
- Actionable insights, not just summaries
- Each takeaway should answer: "What can I do with this?"
- Use clear, memorable language

### 4. Chapter/Part Summary
- Brief overview of each major section
- Key concept from each chapter
- Keep each summary to 2-3 sentences

### 5. Practical Applications
- How to apply these ideas in daily life
- Specific examples and scenarios
- Common mistakes to avoid

### 6. Memorable Quotes (3-5)
- Select quotes that capture essential ideas
- Brief context for each quote

### 7. Connections
- Related books or ideas
- How this fits into broader thinking
- Complementary or contrasting perspectives

### 8. Personal Reflection Questions
- 3-5 questions to help internalize the content
- Questions that prompt action, not just thinking

### 9. One-Page Summary
- If you had to explain this book in 60 seconds, what would you say?
- The "cocktail party" explanation

## Writing Style
- Clear, conversational language
- Avoid jargon unless essential (then explain it)
- Focus on understanding and application, not documentation
- Write as if explaining to a curious friend

## Output
Save to: ./books/[book-name-slug].md
